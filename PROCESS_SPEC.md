# 우유 생산공정 공통 조건 (대시보드 단일 기준)

이 문서는 **대시보드 모든 페이지에서 동일하게 유지해야 하는 공정/배치 정의**를 정리합니다.  
코드의 단일 소스는 `haccp_dashboard/lib/process_spec.py` 입니다.

## 1) 공정 구조(현실적 HACCP 관점)

- 생산 라인: 총 3개 (Line 1~3)
- 운영 기간: 총 5일
- 일일 가동: 20시간
- 공정은 연속적으로 흐르지만, **품질관리/추적을 위해 2시간 단위 time-based lot을 “배치(batch)”로 정의**합니다.

### 배치(time-based lot) 정의 (절대 원칙)

- 배치 = “반복 공정 횟수”가 아니라, **특정 2시간 동안 동일 조건에서 생산된 제품 묶음(시간 구간)** 입니다.
- 배치 하나는 아래 공정 단계를 **단 1회만** 순차적으로 통과합니다(반복 없음).

공정 단계(배치당 1회, 순차):
원유입고 → 저장 → 여과 → 표준화 → 가열 → 보온 → 냉각 → 충진 → 검사 → 출하판정

금지 표현:
- “배치 하나가 하루에 여러 번 공정을 돈다” (공정 반복 ≠ 배치 반복)

## 2) 전체 배치(150) 계산 논리

- 2시간 × 10회 = 라인당 하루 10개 배치 생성
- 3개 라인 × 하루 10배치 × 5일 = 총 150배치
- 모든 배치는 시간 순서에 따라 생성되며, 라인은 병렬 운영됩니다.

## 3) 데이터 구조(대시보드 설계에 반영)

대시보드는 배치를 추적 단위로 삼기 위해, 데이터에 최소한 아래 키를 일관되게 유지합니다.

- `batch_id`: time-based lot(2시간) 단위 배치 식별자
- `line_id`: 생산 라인(1~3)
- `line_day`: 운영일(1~5)
- `line_run`: 해당 운영일 내 2시간 lot 순번(1~10)
- `datetime`: 측정 시각(연속공정의 흐름을 유지)
- `state`: 공정 단계(위 10개 단계 중 하나)

## 4) 대시보드 페이지 설계(일관성 원칙)

- 모든 페이지는 동일한 공정 단계 정의(10단계, 단일 패스)와 batch 정의(2시간 lot)를 사용합니다.
- “실시간”은 연속 데이터 스트림이지만, 화면의 요약/비교/알람은 **배치 단위(2시간 lot)** 로 집계합니다.
